3. MainActivity3.kt (Data Summary & Firebase Read)

class MainActivity3 : AppCompatActivity() {
    override fun onCreate(...) {
        // ... (standard setup)
Standard Activity start function for the Summary Screen.
        val name = intent.getStringExtra("NAME") ?: ""
Receives the unique ID (the name) passed from MainActivity2. If nothing was passed, it safely defaults to an empty string.
        FirebaseDatabase.getInstance().getReference("firebase crude operation")
            .child(name).get().addOnSuccessListener { snapshot ->
The Firebase Read Operation: Queries the precise location inside "firebase crude operation" using the `name` as the key. `.get()` requests a one-time snapshot of the data. `.addOnSuccessListener` means "once you successfully pull the data down from the cloud, run the code block below."
            findViewById<TextView>(R.id.textViewDisplayName).text = "Name: " + snapshot.child("name").value.toString()
            findViewById<TextView>(R.id.textViewDisplayAge).text = "Age: " + snapshot.child("age").value.toString()
            findViewById<TextView>(R.id.textViewDisplayGender).text = "Gender: " + snapshot.child("gender").value.toString()
            findViewById<TextView>(R.id.textViewDisplayBirthdate).text = "Birthdate: " + snapshot.child("birthdate").value.toString()
        }
Inside the success block: `snapshot` contains everything Firebase sent back. We access individual fields inside the snapshot (like `age`, `gender`), convert their raw values back into strings, and inject them directly into the visual TextView elements on the screen.
        findViewById<Button>(R.id.buttonBack).setOnClickListener {
            startActivity(Intent(this, MainActivity::class.java).apply { flags = Intent.FLAG_ACTIVITY_CLEAR_TOP })
        }
Sets a click listener on the Back button. It launches MainActivity (the login screen) and utilizes `FLAG_ACTIVITY_CLEAR_TOP` to destroy all intermediate screens, bringing you cleanly back to square one without building up memory.
